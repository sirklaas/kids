App Navigation Architecture

Three approaches to structuring the 7-stage flow — which feels right?

A

Spec-Faithful Hybrid — Recommended

Stages 1–4 live on a single dashboard page with progressive reveal (each stage appears below the last as you advance). Stages 5, 6, 7 are full-page routes (/plotboard, /story, /prompts) exactly as the spec describes.

✓ Matches spec intent exactly   ✓ Focused full-page views for complex stages   ✓ Clean URL bookmarking for Stages 5–7
B

All Routes

Every stage is its own route: / → /titles → /synopsis → /plotboard → /story → /prompts. Browser back/forward works at every step. Progress tracked in PocketBase.

✓ Cleanest routing   ~ More navigation boilerplate   ✗ Loses "stages appearing inline" feel of Stages 1–4
C

Full SPA (Single Page)

All 7 stages render on one page. Active stage is shown, others hidden. A top stepper bar shows progress. No route changes at all.

✓ Simplest routing   ✗ Stages 5–7 feel cramped without full-page treatment   ✗ No shareable deep links
Route map for Option A
